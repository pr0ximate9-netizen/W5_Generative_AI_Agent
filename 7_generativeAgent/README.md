# SeatWise Agent Simulation

## 프로젝트 소개

본 프로젝트는 컴퓨터 실습실 좌석 부족 문제를 다루는 Multi-Agent Simulation 과제입니다.

현재 대학 생활을 하고 있는 학생의 관점에서, 주변 학생들이 겪을 수 있는 문제 상황을 대입하여 기획해 보았습니다.

AI, 프로그래밍, 데이터분석 과목 등 수강생이 늘어나면서 컴퓨터 실습실 이용 수요도 함께 증가하고 있습니다. 
학생은 실습실에 가도 자리를 찾지 못해 과제 수행에 어려움을 겪고, 
조교는 수업 운영과 혼잡 관리에 부담을 느낍니다. 
교수는 교육 품질 저하를 걱정하고, 
실습실 관리자는 제한된 예산 안에서 공간을 효율적으로 운영해야 합니다.

SeatWise는 이 문제를 한 사람의 불편이 아니라 여러 이해관계자가 함께 논의해야 하는 커뮤니티 문제로 보고, 학생, 조교, 교수, 관리자, 총학생회 Agent가 토론하며 해결책을 찾는 과정을 구현했습니다.

## 기획 의도

컴퓨터 실습실 좌석 부족은 단순히 좌석 수가 적어서 생기는 문제가 아닙니다.

- 시험기간에 이용자가 몰리는 경우
- 오후 2시부터 6시 사이에 사용자가 집중되는 경우
- 가방만 놓고 자리를 차지하는 경우
- 실습 과목 증가로 수업 시간과 자율 이용 시간이 겹치는 경우
- 예산 문제로 실습실을 바로 늘리기 어려운 경우

따라서 본 시뮬레이션에서는 서로 다른 입장을 가진 Agent가 대화를 나누고, 대화 내용을 바탕으로 Memory와 Reflection을 갱신한 뒤, 마지막에 Insight Analyzer가 정책 제안 형태의 리포트를 생성하도록 구성했습니다.

```text
의견 수집 -> 토론 -> Reflection -> 정보 축적 -> 인사이트 생성 -> 정책 제안
```

## 구현 내용

### Agent

각 Agent는 다음 정보를 가집니다.

- 이름
- 나이
- 역할
- 성격
- 초기 기억
- Memory
- Reflection
- 하루 계획

학생 Agent는 좌석 부족과 긴 대기시간을 중요하게 생각하고, 관리자는 예산과 공간 활용률을 우선적으로 고려합니다. 이렇게 Agent별 배경을 다르게 설정하여 같은 문제를 보더라도 서로 다른 관점에서 말하도록 만들었습니다.

### Planning

시뮬레이션이 시작되면 각 Agent는 하루 동안 어떤 장소에서 무엇을 할지 간단한 계획을 갖습니다.

예시 공간:

- 컴퓨터 실습실 A
- 컴퓨터 실습실 B
- 강의실
- 학과 사무실
- 학생 회의실

예시 일정:

```text
09:00 @ 컴퓨터 실습실 A: 오늘의 좌석 이용 상황을 확인한다.
11:00 @ 컴퓨터 실습실 B: 자신의 역할 관점에서 문제를 정리한다.
14:00 @ 강의실: 혼잡 시간대와 자리 점유 문제를 공유한다.
16:00 @ 학과 사무실: 예약제와 자동 취소 방안을 검토한다.
18:00 @ 학생 회의실: 토론 내용을 정리하고 정책 제안을 준비한다.
```

### Conversation

같은 주제를 두고 Agent들이 라운드별로 자동 대화를 진행합니다.

Round 1에서는 주로 문제 상황을 공유합니다.

```text
좌석 부족 경험, 혼잡 시간대, 무단 자리 점유 문제 공유
```

Round 2에서는 해결책을 논의합니다.

```text
좌석 예약 시스템, 미사용 자동 취소, 수업 시간표 분산 방안 논의
```

현재 버전은 API 키 없이 바로 실행할 수 있도록 미리 정의한 발화를 사용합니다. 그래서 실행 환경이 달라도 같은 결과를 안정적으로 확인할 수 있습니다.

### Memory

대화가 끝나면 각 Agent는 다른 Agent가 말한 내용을 사실 형태로 저장합니다.

예를 들어 조교가 다음과 같이 말하면,

```text
점심 이후, 특히 오후 2시부터 6시 사이에 이용자가 급격히 늘어납니다.
```

다른 Agent들은 조교에 대해 다음 사실을 기억합니다.

```text
조교는 '점심 이후, 특히 오후 2시부터 6시 사이에 이용자가 급격히 늘어납니다.'라고 말했다.
```

### Reflection

Memory가 쌓이면 Agent는 다른 Agent에 대한 간단한 관계 요약을 만듭니다.

예시:

```text
관리자는 자신의 관점에서 문제를 설명하는 사람으로 보인다.
```

이 단계는 단순히 대화 로그를 저장하는 것에서 끝나지 않고, Agent가 다른 Agent를 어떻게 이해하게 되었는지 보여주기 위한 부분입니다.

### Insight Analyzer

시뮬레이션이 끝나면 전체 대화를 바탕으로 핵심 문제, 원인, 해결책, 예상 효과를 정리합니다.

출력 예시:

```text
[문제]
1. 시험기간 좌석 부족
2. 오후 2시부터 6시 사이 특정 시간대 집중 이용
3. 가방만 놓는 무단 자리 점유

[원인]
- AI, 프로그래밍, 데이터분석 실습 과목 증가
- 좌석 예약 시스템 부재
- 실시간 좌석 상태를 확인하기 어려운 운영 방식

[해결책]
1순위: 좌석 예약 시스템
2순위: 30분 이상 미사용 시 자동 취소 기능
3순위: 실습 과목 시간표 분산
4순위: 추가 실습실 확보

[예상 효과]
- 좌석 이용률 증가
- 대기시간 감소
- 학생 만족도 향상
- 실습 수업 운영 안정화
```

### Report

리포트에는 다음 내용이 포함됩니다.

- 특정 Agent 관점의 하루 계획
- 다른 Agent에 대해 알고 있는 사실
- 다른 Agent에 대한 Reflection
- Insight Analyzer 결과

## Agent 구성

### 학생

컴퓨터 실습실을 자주 이용하는 컴퓨터공학과 학생입니다.

주요 관심사:

- 좌석 부족
- 긴 대기시간
- 시험기간 혼잡
- 예약 시스템 도입

### 조교

프로그래밍 실습 수업을 운영하는 조교입니다.

주요 관심사:

- 실습 수업 운영
- 오후 2시부터 6시 사이 혼잡
- 예약 후 미사용
- 무단 자리 점유

### 교수

AI와 데이터분석 과목을 담당하는 교수입니다.

주요 관심사:

- 학생 학습권
- 실습 수업 품질
- 실습 과목 증가
- 시간표 분산

### 관리자

컴퓨터 실습실 운영을 담당하는 관리자입니다.

주요 관심사:

- 예산
- 공간 활용률
- 좌석 상태 확인
- 미사용 좌석 자동 취소

### 총학생회

학생 의견을 수집하고 학교에 전달하는 총학생회 담당자입니다.

주요 관심사:

- 학생 만족도
- 설문조사 결과
- 공정한 좌석 이용
- 예약 시스템

## 파일 구조

```text
7_generativeAgent/
├─ config/
│  ├─ preset.py
├─ scripts/
│  ├─ agent.py
│  ├─ app.py
│  └─ config.py
├─ README.md
├─ main.py
├─ requirements.txt
└─ simulation.py
```

주요 파일 설명:

- `config/preset.py`: Agent, 장소, 시간대, 라운드 주제 등 기본 설정값
- `scripts/agent.py`: Agent, World, 대화, Memory, Reflection, Report, 저장/불러오기 로직
- `scripts/app.py`: Streamlit 화면
- `main.py`: 콘솔 실행 진입점
- `simulation.py`: 기본 시뮬레이션 실행 파일
